## 🔧 Drill Data Management System

### **Comprehensive QA Validation Rules**

#### **1. Interval Overlap Detection**
- **Type**: `interval_overlap`
- **Severity**: Error
- **Description**: Detects overlapping intervals within the same drill hole
- **Parameters**: Configurable tolerance (default: 0.01m)
- **Impact**: Prevents data integrity issues in geological logging

#### **2. Duplicate Hole ID Detection**
- **Type**: `duplicate_hole`
- **Severity**: Error
- **Description**: Identifies duplicate hole IDs in collar table
- **Impact**: Ensures unique identification of drill holes

#### **3. Missing Survey Data**
- **Type**: `missing_survey`
- **Severity**: Warning
- **Description**: Flags holes without downhole survey data
- **Impact**: Identifies potential issues with hole trajectory calculations

#### **4. Invalid Coordinates**
- **Type**: `invalid_coordinates`
- **Severity**: Error
- **Description**: Validates coordinate ranges and elevation limits
- **Parameters**: Longitude/latitude bounds, elevation limits
- **Impact**: Ensures spatial data integrity and prevents mapping errors

#### **5. Below Detection Limit Flagging**
- **Type**: `below_detection_limit`
- **Severity**: Info
- **Description**: Identifies assay values below analytical detection limits
- **Impact**: Helps with statistical analysis and grade estimation

#### **6. Missing Total Depth**
- **Type**: `missing_total_depth`
- **Severity**: Warning
- **Description**: Flags holes without total depth information
- **Impact**: Ensures complete drilling information for analysis

#### **7. Extreme Assay Values**
- **Type**: `extreme_assay_values`
- **Severity**: Warning
- **Description**: Statistical outlier detection using Z-scores
- **Parameters**: Z-score threshold (default: 3.0)
- **Impact**: Identifies potentially erroneous analytical results

### **Quick Start - Complete System**

1. **Clone the repository:**
```bash
git clone https://github.com/geovision/ai-miner.git
cd geovision-ai-miner
```

2. **Install dependencies:**
```bash
# Frontend dependencies (includes all required packages)
npm install

# Backend dependencies (for data processing)
pip install -r api/requirements.txt
pip install -r tools/stac/requirements.txt
```

3. **Environment configuration:**
```bash
# Copy environment template
cp .env.example .env.local

# Configure required variables
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
VITE_MAPBOX_TOKEN=your_mapbox_token
```

4. **Database setup with sample data:**
```bash
# Start Supabase local development
supabase start

# Seed database with 2,650+ sample records
npm run seed

# Or apply the sample data migration
supabase db push
```

5. **Start development server:**
```bash
# Start frontend with sample data
npm run dev

# Access the application at http://localhost:8080
```

```bash

Content-Type: multipart/form-data

# Parameters:
# - file: CSV or Excel file
# - data_type: collar, survey, interval, or assay
# - project_id: Optional project ID
```

#### **QA Validation**
```bash
# Start QA validation
POST /api/v1/drill-data/qa/validate
{
  "project_id": "uuid",
  "hole_ids": ["DDH001", "DDH002"]
}

# Check validation status
GET /api/v1/drill-data/qa/validate/status/{task_id}

# Get QA results with filtering
GET /api/v1/drill-data/qa/results?severity=error&status=open&limit=100

# Resolve QA issue
PUT /api/v1/drill-data/qa/results/{result_id}/resolve
{
  "resolution_note": "Issue manually verified and resolved"
}
```
